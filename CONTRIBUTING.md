# Mitwirken bei Open-Akquise

Vielen Dank, dass du dich für eine Mitarbeit an **Open-Akquise** interessierst! Wir freuen uns über jede Unterstützung, sei es durch das Melden von Fehlern, das Vorschlagen neuer Funktionen oder das Einreichen von Code-Änderungen.

## Wie kann ich beitragen?

### 1. Fehler melden (Bug Reports)
Wenn du einen Fehler findest, erstelle bitte ein **Issue** im GitHub-Repository. Beschreibe das Problem so detailliert wie möglich:
- Was hast du getan?
- Was hast du erwartet?
- Was ist stattdessen passiert?
- Screenshots helfen immer!

### 2. Feature-Vorschläge
Du hast eine Idee für eine neue Funktion? Super! Erfasse auch dies als ein Issue und kennzeichne es als "Feature Request".

### 3. Pull Requests (Code-Beiträge)
Wenn du Code beisteuern möchtest, folge bitte diesem Workflow:
1. Erstelle einen eigenen **Branch** für deine Änderungen (z.B. `feature/mein-neues-feature`).
2. Implementiere deine Änderungen.
3. Achte auf sauberen Code und dokumentiere neue Funktionen kurz im Wiki.
4. Erstelle einen **Pull Request** gegen den `main` Branch.

## Entwicklungs-Richtlinien
- Wir nutzen **Next.js 15+** mit dem App-Router.
- Styling erfolgt über **Tailwind CSS 4**.
- Achte auf Barrierefreiheit und Responsive Design (Mobile First).
